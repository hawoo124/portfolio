주요 기획 항목

📌 문제점 진단 및 원인 분석

운영팀 인터뷰 및 CS 데이터 기반으로, 식단 이용 고객의 두가지 이탈 유형을 발견

case 01. 사용자 인식 문제

식단 상품이 월령별 성장단계에 따라 연속적으로 이루어지는 구조이지만 사용자는 식단을 단발성 상품으로 인식

→ 식단 지속 혜택이 없어 재구매 동기가 없음.

case 02. 구조 문제

재구매 의향이 있는 사용자도 다음단계 식단을, 탐색 재주문 과정에서 이탈 혹은 CS 문의가 빈번히 발생

→ 식단 지속 혜택이 없어 재구매 동기가 없음.

🎯 해결 방향
  • 사용자 인식 전환 설계 구독 시 다음 단계 식단 안내 및 식단 단계에 따른 아동 발달 과정까지 구체적으로 안내 → 식단을 '여정'으로 인식하도록 리프레이밍
  • 구독 유지 혜택 설계 '다음 단계 맛보기', '구독 전용상품' 등 혜택 설계 → 서비스 이용 동기 제공
  • 구독 결제 프로세스 빌링키 기반 자동 결제 프로세스 설계 → 단계 전환 및 결제 자동 지원
✅ 기대 효과
  • 식단 재구매율 상승구매 자동 전환으로 이탈 방지
  • 고객 LTV 향상구독 모델로 지속적인 매출 확보
  • 락인 효과구독 효과를 경험한 사용자는 혜택 경험을 통해 락인 가능성 ↑
📌 문제점 진단 및 원인 분석

검색 기능 자체가 부재했지만, 단순히 검색창을 추가하는 것으로는 해결되지 않는 특수성 존재

01. 복잡한 상품명

재료 조합으로 구성된 긴 명칭이 대부분이고, 유사한 명칭의 상품이 여러 카테고리에 분산되어 있어 키워드 검색을 통한 탐색이 어려움 (ex. 중기 - 한우연근애호박죽, 후기 - 한우대추흑미죽)

02. 조건기반 탐색

사용자 탐색 의도가 '아이 월령에 맞는 상품', '특정 재료가 포함된 식단' 등 조건 기반 탐색에 가까움.

03. 사용자 탐색 데이터 부재

판매량, PV 외에 사용자가 실제로 무엇을 찾는지 알 수 없어 '진짜 니즈' 파악이 어려움.

🎯 해결 방향

💡 3축 검색(키워드/카테고리/재료) + 인기검색어
다양한 방식의 검색 구조와 인기 검색어를 노출함

  • 키워드 검색 상품명의 일부 및 카테고리 위치를 아는 사용자가 직접 탐색
  • 카테고리 검색 월령 및 식단 유형 기반 카테고리 검색
  • 재료 검색 상품명을 몰라도 포함된 식재료로 검색 가능
  • 검색 데이터 축적하여 인기 검색어 노출 · 사용자: 인기검색어 노출로 탐색 비용 감소
    · 운영자: 실수요 파악 데이터로 활용
✅ 기대 효과
👩 사용자
  • 탐색 성공률 향상다양한 방법으로 원하는 상품에 도달
  • 탐색 비용 감소인기 검색어 제공으로 직접 탐색하지 않아도 상품 발견 가능
👩‍💻 운영자
  • 사용자 실수요 파악검색어 데이터로 사용자의 직접적인 탐색 의도 확인 가능
  • 상품 기획 데이터 확보검색 수요 기반으로 신규 상품 및 의사 결정 데이터로 활용 가능
📌 문제점 진단 및 원인 분석

식단 관련 CS 문의 과다 → 운영 부담 과중 + 사용자 위험 이탈

01. 시스템 제약

  • 메뉴 커스텀 불가 - 선호/기피 재료 반영 불가
  • 단계 변경 불가 - 아이 식단
  • 개별 회차 일정변경 불가 - 1회차 변경 시 전체 일정 연동 변경

→ 전부 CS 처리하여 부담 가중


02. 사용성이 저하된 UX

  • 메뉴, 배송일, 배송지 변경 기능이 분산되어 있어 기능 탐색보다 CS 문의로 의존함.
  • 알러지 성분을 한 번에 확인하기 어려워 알러지 식재료 포함 메뉴 수령 후 환불 요청
🎯 해결 방향

💡 사용자가 컨트롤 할 수 있는 식단 플래너
CS에 의존하지 않고 사용자가 하나의 메뉴에서 식단을 컨트롤 할 수 있도록 변경

  • UX 단일 메뉴 관리 메뉴, 배송일, 배송지, 단계 변경을 식단플래너 내 하나의 화면으로 통합
  • 기능 식재료 기반 메뉴 필터링 배송 예정 메뉴를 식재료로 필터링하여 개인화된 식단으로 커스텀 가능
  • 개인화 알러지 개인화 경고 개인 알러지 정보를 등록하여 배송 예정 메뉴에 성분 포함 시 UI 경고
✅ 기대효과
👩 사용자
  • 자기 해결률 향상CS 의존없이 직접 식단 컨트롤 가능
  • 식단 만족도 향상 및 이탈률 감소단계 변경 가능, 알러지 사전 안내로 만족도 증가 및 클레임/환불 감소
👩‍💻 운영자
  • 운영부담 감소식단 관련 운영 문의 감소로 운영 비용 절감 가능
  • 맞춤형 추천 데이터 확보개인 알러지 정보를 통해 맞춤형 상품 추천 가능

    • 단계별 상품 연계 및 추천 서비스 구현 가능
    • 구독 서비스의 다음 단계 추천 가능
  • UX FO 상세페이지 핵심 정보 집약 최상단에 가격, 혜택, 쿠폰 정보 집약 배치

Project 02

어드민 고도화
— 프로모션 페이지 자동화

연 1,084건의 프로모션을 매번 하드코딩으로 처리하던 구조를, 어드민 단일 메뉴에서 자동화 운영할 수 있도록 시스템화했습니다. 연 2.17억원의 개발 비용을 절감하고 일정 리스크를 90% 이상 해소했습니다.

기여도 기획 70% / 전체 기획 100%

문제 정의 및 해결 방향

⚠ AS-IS 문제

  • 프로모션 조건·혜택·노출 요소가 여러 메뉴에 분산되어 운영 복잡도 높음
  • 기획·디자인·개발이 순차적으로 진행되어 요건 지연 시 오픈 연기
  • 하드코딩으로 수정 발생 시 재배포 필요 → 프로모션 일시 중단
  • 연 1,084건 × 1~1.5M/D 개발 공수 발생

✅ TO-BE 개선

  • 프로모션 관리 메뉴 단일화: 조건·혜택·노출을 하나의 화면에서 설정
  • 프로모션 코드 선연동 방식: 요건 확정 전 개발 선착수 가능
  • 운영 중 수정 실시간 반영: 재배포 없이 즉시 적용
  • 운영 자립화: 프로모션이 '개발 과제'에서 '운영 과제'로 전환

기대 효과 상세

💰 개발 비용 절감

프로모션당 개발 투입 시간 1~1.5MD 단축 → 약 51.62MM 단축. 연 2.17억원 절감 (1,084건 × 1MD × 약 20만원 기준).

⏱ 기획 공수 절감

전체 프로모션 중 70% 이상 기획 투입 감소 → 약 18.06MM 단축.

📅 일정 리스크 제거

요건 확립 전 개발 선착수 가능. 일정 변경 리스크 90% 이상 감소.

🚀 운영 자립화

신규 프로모션 유형 자체 대응. 타겟 세분화·혜택 다양화 등 다양한 유형 독립 운영 가능.

Project 03

메인 PC UX 개선

고객 이탈 원인을 GA4·Beusable 행동 데이터로 분석하고, Average Fold 내 핵심 콘텐츠 재배치와 어드민 연동을 통해 페이지뷰 131%, 체류시간 170%, 전환율 18.7%를 향상시켰습니다.

기간 2022.05 – 2022.07
역할 PM / 기획 / QA
기여도 프로젝트 50% / 기획 100%

주요 성과

131%
방문자당 페이지뷰 증가
170%

+18.7%
전환율 증가

문제 정의 (GA4 · Beusable 기반)

📉 주요 상품 미노출

Average Fold(991px) 이내 주요 상품 노출 없음. 약 53% 이상이 해당 영역 이전에 이탈.

📉 낮은 체류·전환 지표

UV 대비 PV·체류시간이 낮음. 상품 페이지 이탈이 83%에 가깝지만 후킹 요소 부족.

🔀 비효율적 UX 플로우

메인 아이템 배리에이션 구조가 타겟 유저와 불일치. 무효 클릭 빈번 발생.

⚙ 운영 비효율

대부분의 영역이 하드코딩되어 콘텐츠 즉각 반영 불가. 디자인·개발 의존도 높음.

해결 방향

⚠ AS-IS

  • 접힘 상단 영역에 상품 미노출 → 이탈률 53%
  • IA 미정립으로 탐색 비효율 및 무효 클릭 빈번
  • 상품 노출 연계 없어 이탈·전환율 저조
  • 메인 운영 영역 하드코딩 → 즉각 수정 불가

✅ TO-BE

  • Average Fold 상단에 주력 상품·콘텐츠 배치 → 이탈률 25%로 감소
  • IA 정립으로 탐색 동선 최적화 → 이탈률 88%p 개선
  • 콘텐츠 게시물에 연관 상품 연동으로 전환율 향상
  • 반복 운영 영역 어드민 연동 → 즉각 콘텐츠 반영 가능

성과 해석

📄 PV 131.2% 향상

방문자당 페이지뷰 수 증가로 사용자에게 유의미한 콘텐츠가 효과적으로 배치되었음을 확인.

⏱ 체류시간 170% 향상

평균 체류시간 증가로 사용자에게 흥미로운 콘텐츠가 늘었음을 추측할 수 있음.

💳 전환율 18.7% 향상

전환율 증가로 사이트 내 상품 노출이 효과적으로 진행된 것을 확인.
